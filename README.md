The core objective of this project is to demonstrate:

Test-Driven Development (TDD) principles.

Validation of complex logic using JUnit 5.

Clean code practices and efficient algorithm design.

🛠 Tech Stack
Language: Java

Testing Framework: JUnit 5

Build Tool: Maven/Gradle (as applicable)

IDE: IntelliJ IDEA / Eclipse

🚀 Key Features & Algorithms
The project includes Java implementations for several high-level logic problems, each accompanied by comprehensive test coverage:

Matrix Operations: Advanced traversal logic (e.g., Spiral Order) with boundary validation.

String Manipulation: Palindrome detection and anagram grouping.

Data Structures: Custom implementations of caches and linked lists.

🧪 Testing Rigour
Every class in this repository is paired with a corresponding Test class. My testing strategy focuses on:

Positive Testing: Validating standard expected behavior.

Negative Testing: Ensuring the system handles null or invalid inputs gracefully.

Boundary Analysis: Testing the "edges" of data structures (empty lists, single elements, max integers).
